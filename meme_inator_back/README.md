# Project tree вЂ” one Django app per API tag from OpenAPI spec, each with `delivery / domain / infrastructure`

Every app follows the same micro-module pattern:

* `delivery/` вЂ” Django-Ninja routers, request/response schemas, Channels consumers, GraphQL resolvers (edge adapters)
* `domain/` вЂ” pure business logic: entities, value objects, use-cases, domain services, repository interfaces (ports)
* `infrastructure/` вЂ” framework/IO adapters: Django ORM models, repository implementations, external API adapters, mappers, background tasks

There is also a include a small shared `core/` for DI, result types, and auth, plus a top-level `infrastructure/shared/` for S3/redis/celery clients that multiple apps can reuse.

# Responsibilities for each folder (short)

* `delivery/api/routers.py` вЂ” define Ninja `Router()` and endpoints. Use pydantic schemas (`schemas.py`) here.
* `delivery/schemas.py` вЂ” request/response DTOs, validation.
* `delivery/consumers.py` вЂ” Channels consumers (async) that call domain usecases.
* `domain/entities.py` вЂ” dataclasses or simple objects representing domain, no Django imports.
* `domain/usecases/*.py` вЂ” one file per use-case (execute method + pure logic orchestration). Accept repository interfaces.
* `domain/interfaces.py` вЂ” abstract repositories (ABCs) referenced by usecases.
* `infrastructure/models.py` вЂ” Django ORM models (import only Django).
* `infrastructure/repositories.py` вЂ” concrete implementations of `domain.interfaces` using ORM/mappers.
* `infrastructure/mappers.py` вЂ” translate ORM <-> domain objects (keeps domain pure).
* `infrastructure/*_adapter.py` вЂ” external adapters (S3, push, search).
* `core/di.py` вЂ” bind interfaces to infra implementations during app startup (call binder in `memeinator/asgi.py` or relevant AppConfig.ready()).

# Minimal example flow (where code lives)

Create post flow:

1. Request hits `apps/posts/delivery/api/routers.py:create_post` (Ninja)
2. Router validates using `delivery/schemas.py::CreatePostSchema`
3. Router calls `domain/usecases/create_post.py::CreatePostUseCase` with injected `IPostRepository` & `IMediaService`
4. Usecase applies domain rules, calls repo interface methods
5. `infrastructure/repositories.py::DjangoPostRepository` persists using ORM `models.py`
6. Router returns `schemas.PostOut` to client
